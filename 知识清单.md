# ClawAgent 项目知识点清单

## 1. LLM / AI

### 框架与库
- **langchain** (>=1.3.11) — `create_agent` ReAct Agent 构建；`init_chat_model` 多 Provider 模型初始化
- **langchain-core** (>=1.4.8) — `BaseTool`、`@tool` 装饰器、`BaseMessage`、`BaseChatModel`、`BaseCallbackHandler`
- **langchain-anthropic** (>=1.4.7) — Anthropic Provider 集成；`AnthropicPromptCachingMiddleware` 瞬态 Prompt 缓存（TTL=5min）
- **langchain-openai** (>=1.0.0) — OpenAI 兼容 Provider（SiliconFlow Worker 模型用）
- **langgraph** (>=1.2.6) — `CompiledStateGraph`、`SqliteSaver` checkpointer、`invoke()`/`stream()` 状态图执行
- **langgraph-prebuilt** — `create_react_agent` 快速创建工具调用 Agent
- **langgraph-checkpoint-sqlite** (>=3.1.0) — SQLite 检查点持久化
- **langchain-community** — 社区 Provider 集成
- **pydantic** — `SecretStr` 安全存储 API Key

### AI 技术 & 模式
- **ReAct Agent 模式** — `langchain.agents.create_agent` 工具调用循环
- **多 Provider 模型初始化** — `init_chat_model` 运行时切换 Provider（anthropic / openai / deepseek / ollama 等）
- **Prompt Caching 中间件** — `AnthropicPromptCachingMiddleware`，`unsupported_model_behavior="ignore"` 兼容 DeepSeek
- **before_model 中间件 Hook** — 在每次模型调用前注入上下文压缩
- **混合 Token 估算** — ASCII 字符 4 chars/token，CJK 字符 2 chars/token
- **Token 用量追踪** — `TokenCounter` LangChain 回调，从 `usage_metadata`（ChatAnthropic）和 `response_metadata`（DeepSeek）提取
- **LLM 摘要** — 用 LLM invoke 生成对话摘要（上下文压缩 + 会话记忆）
- **LLM 偏好提取** — 用 LLM + JSON 结构化输出从对话中提取用户偏好

### 第三方 AI 服务
| 服务 | 端点/协议 | 用途 |
|------|----------|------|
| DeepSeek | Anthropic 兼容 API (`api.deepseek.com/anthropic`) | 主 Agent（deepseek-v4-flash/pro） |
| SiliconFlow Embedding | REST API (`api.siliconflow.cn/v1/embeddings`) | 云端嵌入 `Qwen/Qwen3-VL-Embedding-8B`（768 维） |
| SiliconFlow Chat | OpenAI 兼容 API | Researcher/Critic Worker（Qwen 模型） |

## 2. 搜索与检索（RAG）

### 库
- **chromadb** (>=1.2.0) — 持久化向量库 `PersistentClient` + `get_or_create_collection()`
- **jieba** (>=0.42.1) — 中文分词，BM25 词法检索
- **httpx** — Bing 搜索 HTTP 抓取
- **beautifulsoup4** — HTML 解析（`lxml`，`li.b_algo` 选择器）
- **trafilatura** (>=2.1.0) — 网页正文提取（去除广告/导航/侧栏）

### 检索技术
- **KNN 向量检索** — Chroma 向量库 + SiliconFlow 嵌入，距离转分数 `1/(1+distance)`
- **BM25 词法检索** — 内存索引 + jieba 分词，k1=1.5, b=0.75
- **混合检索 RRF 融合** — Reciprocal Rank Fusion（k=60），MD5 去重（前 100 字符）
- **后台 BM25 构建** — daemon 线程后台构建，构建期间降级为纯 KNN
- **BM25 缓存** — Pickle 序列化 + SHA256 语料哈希校验
- **批量嵌入** — 每批 100 条文本，指数退避重试（最多 3 次）

### 数据格式
- **Pickle** — BM25 索引缓存
- **SHA256** — 语料内容哈希缓存校验

### 外部服务
- **Bing 搜索** — `https://www.bing.com/search?q=...&count=10`，3 次重试，2s 延迟，10s 超时

## 4. 数据与存储

### 数据库
- **SQLite** — 主存储：会话摘要、对话消息、用户偏好、LangGraph 检查点
- **WAL 模式** — `PRAGMA journal_mode=WAL` 并发读写
- **row_factory** — `sqlite3.Row` 字典式行访问
- **ChromaDB PersistentClient** — 磁盘向量库 `chroma_db/`

### 数据格式
- **JSONL** — 按会话日志 `logs/{thread_id}.jsonl`
- **TOML** (`tomllib`) — 价格配置 `price.toml`，构建系统 `pyproject.toml`
- **Pickle** — BM25 索引序列化
- **Markdown** — Prompt 文件

### SQL 表结构
- `session_summaries` — thread_id(PK), title, summary, message_count, timestamps
- `conversation_messages` — id(auto), thread_id(indexed), role, content, created_at
- `preferences` — id(auto), key(indexed), value, confidence(REAL), session_id, evidence, created_at

### 数据库模式
- **连接缓存** — `_conn_cache` dict + `threading.Lock` 池化连接
- **惰性连接** — `_get_cached_conn` 首次访问时创建
- **check_same_thread=False** — 多线程 SQLite 访问
- **INSERT OR REPLACE / INSERT OR IGNORE** — 幂等 upsert

- **连接缓存锁** — `_cache_lock` 保护 `_conn_cache`

## 7. 多 Agent 编排

### 设计模式
- **Worker 模式** — 独立子 Agent（Coder/Researcher/Critic/Writer），临时创建、任务完成销毁
- **工厂模式** — `WorkerFactory` 按角色按需创建 Worker
- **注册表模式** — `@register_worker` 装饰器填充 `WORKER_REGISTRY`；`import clawagent.worker` 副作用注册
- **委托工具** — `delegate_task` 闭包工具，绑定特定 `WorkerFactory` 实例（无全局状态）
- **隔离 Worker 实例** — 独立模型、Prompt Builder、SQLite Checkpointer、thread_id、工具集

### Worker 角色
| 角色 | 工具 | 默认 Provider |
|------|------|--------------|
| Coder | read_file, write_file, run_command | deepseek-v4-flash (anthropic) |
| Researcher | search_documents, web_search | Qwen via SiliconFlow (openai) |
| Critic | read_file, search_documents | Qwen via SiliconFlow (openai) |
| Writer | read_file, write_file | deepseek-v4-flash (anthropic) |

## 8. API Key 管理

### 库
- **httpx** — `BaseTransport` 子类 HTTP 层 Key 注入 + 401/429 重试

### 设计模式
- **API Key 池** — `ApiKeyPool`，命名池，线程安全 Key 选择，3 种策略（round_robin / random / least_errors）
- **指数退避** — 降级 Key 冷却时间翻倍（起始 30s，最大 600s），冷却后自动恢复
- **Key 状态机** — ACTIVE → DEGRADED (429/5xx) → ACTIVE (冷却后) / ACTIVE → DISABLED (401)
- **传输层拦截** — `KeyPoolTransport` 包装 httpx transport，拦截 429/401，换 Key 重试（最多 3 次）
- **模型层包装** — `KeyPoolChatModel` 扩展 `BaseChatModel`，注入 Key，失败重试（最多 4 次）
- **Token 计数** — 每 Key 用量追踪 via `TokenCounter`
- **全局单例** — `_GLOBAL_POOL` 模块级变量，惰性初始化 `get_global_pool()`
- **环境变量加载** — `load_pools_from_env()` 发现 `API_POOL_{NAME}_KEY_{N}`

## 9. 上下文压缩

### 三种策略
| 策略 | 文件 | 说明 |
|------|------|------|
| trim | `strategies.py:trim_by_count` | 保留系统消息 + 最后 N 条非系统消息（默认 40） |
| token_trim | `strategies.py:trim_by_tokens` | 弹出最旧消息直到估算 token ≤ 阈值（80K） |
| summarize | `strategies.py:summarize_by_llm` | LLM 摘要溢出消息，保留最近 N 条原始消息，合并旧摘要；30s 超时 |

### 模式
- **策略模式** — `compress_state()` 根据 `CompressionConfig.strategy` 分发
- **State modifier 钩子** — `make_state_modifier()` 为 LangGraph `before_model` 创建 pre-model hook
- **摘要合并** — 旧摘要提取后合并到新摘要 Prompt；`[对话历史摘要]` 前缀
- **超时保护** — `summary_timeout` 可配置（默认 30s），通过 `model.invoke(config={"timeout": ...})`

## 10. Prompt 管理

### 文件架构
- **5 层拼装** — 身份 → 人格 → 工作区上下文 → 运行时 → 用户偏好 → 工具列表 → 额外上下文
- **Markdown Prompt 文件** — `prompts/agents/{id}/identity.md`（必需），`soul.md`（可选）
- **共享上下文文件** — `prompts/shared/bootstrap.md`、`agents.md`、`search-rules.md`、`worker-{role}.md`
- **工具列表自动生成** — 遍历 `ALL_TOOLS + delegate_task`，使用 `.name` / `.description`
- **优雅降级** — 缺失 identity.md 生成通用 fallback；缺失可选文件静默跳过

### 模式
- **PromptBuilder** — 可配置构建器，延迟 SQLite 偏好加载
- **惰性导入** — `from clawagent.tools import ALL_TOOLS` 在 `_build_tools_section()` 内（避免循环导入）

## 11. 记忆与偏好

- **会话摘要** — LLM 生成标题 + 摘要存 SQLite
- **对话持久化** — 所有 user/assistant 消息存 `conversation_messages`
- **偏好提取** — LLM（JSON 数组输出）或启发式（字符频率）从对话提取；每 5 轮触发一次（首轮立刻触发）以降低延迟
- **偏好排序** — confidence DESC，限制 top-N（默认 5）
- **启发式 fallback** — model=None 时用基础字符统计（中文检测）

## 13. 流式系统

### 事件类型
`think_start`, `think_end`, `token`, `tool_call`, `tool_result`, `error`, `done`

### 模式
- **类型化事件流** — `StreamEvent` dataclass + `EventKind` Literal
- **状态数据类** — `_StreamState` 封装可变流式状态（all_text, tool_call_accum, in_thinking, usage）
- **四步辅助拆分** — `_process_text_chunk` / `_process_tool_call_chunks` / `_emit_tool_events` / `_extract_usage_fallback`
- **stream_mode="messages"** — LangGraph token 级流式输出
- **Content block 解析** — 处理 DeepSeek 的 thinking/text blocks（`{"type": "thinking", "thinking": "..."}`）
- **增量工具调用组装** — `tool_call_chunks` 在 `tool_call_accum` 累积，node 切换到 "tools" 时 emit
- **Usage 提取 fallback** — `usage_metadata` → `response_metadata` → `graph.get_state()`
- **工具结果预览** — 前 200 字符

## 18. 基础设施

- **SQLite WAL** — `PRAGMA journal_mode=WAL`
- **线程安全** — `check_same_thread=False`，`threading.Lock`
- **Daemon 线程** — BM25 Builder、Agent Producer
- **Signal 处理** — SIGINT 自定义处理器
- **连接生命周期** — `Agent.close()` 清理连接
- **路径沙盒** — `_resolve_path()` 限制在项目根目录
- **Subprocess 超时** — 120s，`TimeoutExpired` 处理
- **API 重试** — 指数退避 (2^n 秒，最多 3 次)
- **API Key 安全** — `SecretStr`，日志排除 Key
- **内存/上下文限制** — 对话截断 50K 字符，页面截断 2K，消息 500 字符
- **热更新** — `Agent.reconfigure()` 不丢失对话状态
